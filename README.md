## 技術スタック

### ランタイム
- **本番**: PHP 8 + cURL（ロリポップ共用サーバー）
- **ローカルデモ**: Node.js（依存ゼロ、標準ライブラリのみ）
- **フロントエンド**: Vanilla JavaScript（フレームワーク不使用）

### AI
- **LLM**: Groq API + `llama-3.3-70b-versatile`
- **代替検証済み**: Claude API（Anthropic）、OpenAI API

### インフラ
- ロリポップ共用サーバー + Cloudflare
- APIプロキシ構成（APIキーはサーバーサイドで完全秘匿）

## AIとの協働プロセス（このプロジェクトの本質）

### 役割分担

| 担当 | 内容 |
|------|------|
| **松島（人間）** | 事業企画・ターゲット定義・価格設計・顧客折衝・意思決定・最終判断・テスト実施 |
| **Claude Code（AI）** | 実装・セキュリティ設計・ドキュメント整形・リファクタリング・テストケース提案 |

### 具体例：Groq + llama-3.3-70b を選んだ意思決定プロセス

「このチャットボットで使うLLM、何がいい？」という議論で、
以下の判断プロセスを踏みました。

1. **最初にコスト試算** — Claude / ChatGPT / Groq の3つで、想定利用量（月1000問い合わせ×50社）のコスト比較をAIに依頼
2. **応答品質の実測** — 日本語FAQ応答の精度をサンプル比較
3. **結論**: Groq + llama-3.3-70b が「低コスト × 爆速応答 × 品質許容」の最適解と判断
4. **運用ポリシー決定**: モデル更新は**無償アップデート範囲内**でのみ実施し、顧客影響を最小化

**この判断はAIが自動で決めたのではなく、事業制約（価格、納品速度、顧客満足度）を加味して人間が決定**しました。

### 具体例：1行で設置 × 商用セキュリティ の両立

「技術者不在の顧客が1行のコードで導入できる」という商品設計上、
セキュリティは**標準装備**として組み込む必要がありました。
AIと対話しながら以下を設計・実装：

| 項目 | 実装内容 |
|------|---------|
| CORS制限 | 顧客ドメインのみ許可（`Access-Control-Allow-Origin` 動的制御） |
| レートリミット | IPベース 10回/60秒（ファイルキャッシュ方式） |
| 入力長制限 | 500文字まで（DoS・コスト攻撃対策） |
| プロンプトインジェクション対策 | SystemPromptでの遵守ルール明記 + 機密情報非開示 |
| HTMLタグ除去 | `<a>` タグはテキスト化、その他はstrip |
| APIキー秘匿 | `.env`読み込み + サーバーサイド完結（ブラウザに渡さない） |
| エラー情報隠蔽 | `error_reporting(0)` + 統一エラーメッセージ |

AIが提案したコードを全パターン実テスト（**28項目 + セキュリティ6項目**）してから本番投入しました。

## セットアップ（ローカルデモ）

### 必要環境
- Node.js 18+ （依存パッケージゼロ、標準ライブラリのみ使用）
- Groq API キー（[console.groq.com](https://console.groq.com/) で無料取得可能）

### 手順

```bash
# 1. クローン
git clone https://github.com/coco888369/asurich-ai-chatbot-demo.git
cd asurich-ai-chatbot-demo

# 2. 環境変数設定
cp .env.example .env
# .env を編集して GROQ_API_KEY を設定

# 3. 起動
node server.js

# 4. ブラウザで http://localhost:3456 を開く
```

## 本番版との差分

このリポジトリは **技術構成の説明用デモ** です。実サービスとの差分は以下：

| 機能 | デモ版 | 本番版 |
|------|--------|--------|
| 基本チャット機能 | ✅ | ✅ |
| システムプロンプト | ✅（公開用・アスリッチ情報） | ✅（各顧客向けカスタム） |
| CORS/レート制限 | ✅ | ✅（顧客ドメイン別） |
| 管理画面 | なし | あり（FAQ差し替え・ログ確認） |
| LINE連携 | なし | あり |
| リッチメニュー | なし | あり（フルプランのみ） |
| GA4 / Clarity計測 | なし | あり |
| 複数サイト対応 | なし | あり（1つのプロキシで複数顧客） |

**顧客保護のため、本番固有のプロンプト設計・FAQデータ・管理画面コードは非公開** としています。

## 事業としての設計思想

このプロダクトは技術ありきではなく、
**「中小企業・個人事業主の問い合わせ対応を、手の届く価格で解決する」** という事業課題から出発しました。

- 顧客の作業は「ヒアリングシートに10分回答」のみ
- FAQ作成・プロンプト設計・設置まで当社で完結
- AIが学習するのではなく、**System Prompt にFAQ情報を構造化して記述**（機械学習ではなく仕組みで解決）
- 買い切り型で月額ゼロ → ランニング負担を下げる
- セキュリティは標準装備 → 顧客にリスクを負わせない

この設計思想は[`docs/workflow.md`](./docs/workflow.md) と [`docs/hearing-sheet.md`](./docs/hearing-sheet.md) に実運用フォーマットとして落とし込まれています。
