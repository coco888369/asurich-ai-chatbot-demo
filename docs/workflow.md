# AIチャットボット構築 受注→納品ワークフロー

> ⚠️ 本ドキュメントは実運用ワークフローの構造を示す**技術デモ用**です。
> 固有の会社名・ドメイン・連絡先等は汎用表記に置き換えています。

## 全体フロー

```
問い合わせ/デモ申込 → ヒアリング → FAQ作成 → 構築 → デモ確認 → 修正 → 本番設置 → 納品・請求
```

---

## Phase 0: 問い合わせ・デモ申込の受領（当日）

### トリガー
- チャットボットLPのフォーム送信
- サイトの問い合わせフォーム
- メッセージングアプリ経由
- 広告経由

### やること
1. 通知メール受信
2. **24時間以内**に一次返信（テンプレ使用）
3. 無料デモ希望の場合 → ヒアリングシート送付
4. 見積もり希望の場合 → 要件確認後にプラン提案

### 一次返信テンプレ（例）

```
○○様

お問い合わせいただきありがとうございます。
[担当者名] と申します。

御社専用のAIチャットボットのデモを無料で作成いたします。

つきましては、添付のヒアリングシートにご回答をお願いいたします。
所要時間は約10分です。ご不明な点は空欄のままでも構いません。

ヒアリングシートをいただきましたら、最短[N日]でデモをお届けいたします。

ご質問がございましたらお気軽にお聞きください。
よろしくお願いいたします。

[担当者名]
[会社名]
[Webサイト URL]
```

## Phase 1: ヒアリング（顧客作業：約10分）

### やること
1. ヒアリングシート（hearing-sheet.md）をPDFまたはGoogleフォームで送付
2. 回答を受領
3. 顧客サイトを確認（サービス内容・料金・FAQ候補の補完）

### 確認ポイント
- [ ] サイトURLが有効か
- [ ] サービス内容・料金が明確か
- [ ] 口調の希望が確認できたか
- [ ] 誘導先（フォーム/電話/LINE）が決まっているか

### プラン確定
ヒアリング内容からFAQ数を見積もり、プランを確定：

| プラン | FAQ数 | 価格（税別） |
|--------|-------|-------------|
| お試し | 〜15問 | 50,000円 |
| ライト | 〜30問 | 80,000円 |
| スタンダード | 〜80問 | 150,000円 |
| フル | 〜150問 | 300,000円 |

## Phase 2: FAQ・回答内容の作成（当社作業：1〜2日）

### やること
1. ヒアリング回答 + 顧客サイト情報をもとにFAQ一覧を作成
2. 各FAQに対する回答文を作成
3. システムプロンプト（chat.php内）を顧客向けにカスタマイズ

### システムプロンプト作成の手順

chat.phpのテンプレを複製し、以下を顧客情報に書き換える：

```
=== 会社概要 ===
社名: {顧客社名}
URL: {顧客サイトURL}
所在地: {顧客所在地}
対応時間: {営業時間}
連絡方法: {誘導先}

=== サービス内容 ===
{ヒアリングシートの回答から作成}

=== 料金 ===
{ヒアリングシートの回答から作成}

=== よくある質問 ===
{作成したFAQ一覧}

=== 回答ルール ===
- {口調の設定}
- 回答範囲外は「{誘導先}」へ案内
- {顧客固有のルール}

=== セキュリティルール ===
{テンプレのまま流用}
```

### 成果物
- FAQ一覧（顧客確認用）
- システムプロンプト（技術実装用）
- チャットウィジェットのカラー設定

## Phase 3: チャットボット構築（当社作業：半日〜1日）

### やること

1. **APIプロキシ（chat.php）のセットアップ**
   - 顧客用のchat.phpを作成（システムプロンプト差し替え）
   - APIキーの設定（採用LLMプロバイダ or 顧客指定のLLM）
   - CORS設定（顧客ドメインのみ許可）

2. **チャットウィジェット（chat-widget.js）の生成**
   - 顧客ブランドカラーに合わせたスタイリング
   - チャットボット名の設定
   - APIエンドポイントの設定
   - 1ファイルにバンドル（顧客が1行貼るだけで動く状態に）

3. **ホスティング**
   - chat.phpを自社サーバー（or 顧客指定サーバー）に設置
   - .envにAPIキーを配置
   - SSL/CORS/レート制限の動作確認

### 技術チェックリスト
- [ ] chat.phpがPOSTリクエストに正常応答するか
- [ ] CORS制限が顧客ドメインのみ許可になっているか
- [ ] レート制限が動作しているか（10回/60秒）
- [ ] APIキーがブラウザから見えないか
- [ ] 500文字制限が効いているか
- [ ] HTMLタグ除去が効いているか
- [ ] プロンプトインジェクション対策が入っているか

## Phase 4: デモ確認（顧客作業 + 当社修正：1〜2日）

### やること
1. デモ用URLまたはテスト環境を顧客に共有
2. 顧客にテスト操作してもらう
3. フィードバック収集（回答内容の修正・追加）
4. 修正対応（FAQ追加・回答文修正・口調調整）

### デモ共有テンプレ

```
○○様

御社専用のAIチャットボットのデモが完成いたしました。

▼ デモURL
{デモURL}

以下の点をご確認いただけますと幸いです：
1. よくある質問に対して適切に回答しているか
2. 料金や営業時間などの情報が正確か
3. 口調やトーンがご希望に合っているか
4. 追加したい質問や修正したい回答がないか

お気づきの点がございましたら、このメールに返信いただくか、
リストにまとめてお送りください。修正いたします。

よろしくお願いいたします。
```

### 修正回数の目安
- プラン内：2回まで無料
- 3回目以降：スポット修正（5,000円/回）として案内

## Phase 5: 本番設置（当社作業：30分〜1時間）

### やること

1. **顧客サイトへの設置コードを生成**

```html
<!-- AIチャットボット by Asurich -->
<script src="https://{ホスト}/chatbot/{顧客ID}/widget.js" defer></script>
```

2. **設置方法を顧客に案内**

```
○○様

チャットボットの本番版が完成いたしました。

▼ 設置方法
以下のコードを、御社サイトの </body> タグの直前に貼り付けてください。

<script src="https://..." defer></script>

WordPressの場合：
外観 → テーマファイルエディター → footer.php の </body> 直前に貼付

設置が難しい場合は、当社で代行いたします（無料）。
サイトの管理画面情報をお知らせいただければ対応いたします。

動作確認後、問題なければ納品完了となります。
```

3. **設置代行の場合**
   - 顧客からCMS管理画面のログイン情報を受領
   - コードを設置
   - 動作確認
   - ログイン情報は作業後に破棄（顧客に通知）

### 本番動作チェック
- [ ] 顧客サイト上でチャットウィジェットが表示されるか
- [ ] 質問に正しく回答するか
- [ ] スマホで正常に動作するか
- [ ] 他のページに遷移しても動作するか
- [ ] ページ読み込み速度に影響がないか

## 注意事項

### LLMバージョン管理
- 現在使用中: 採用LLMプロバイダのモデル（本番環境では非公開）
- バージョンアップ時の方針:
  - **月額保守プラン加入者** → 込みで対応
  - **未加入者** → 1回5,000円で更新
  - **セキュリティ/モデル廃止** → 全顧客に無料対応（必須）

### セキュリティ標準装備（全プラン共通）
- CORS制限（顧客ドメインのみ）
- IPベースレート制限（10回/60秒）
- 入力文字数制限（500文字）
- プロンプトインジェクション対策
- HTMLタグ除去
- APIキー秘匿（サーバーサイドのみ）
- SSL/TLS暗号化（Cloudflare経由）
